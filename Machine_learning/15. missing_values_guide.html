Section 01
How to Identify Missing Values
detection is the first step — before any treatment

Missing values don't always look like blanks. In real datasets they appear in many disguises — NaN, None, empty strings, -1, 999, N/A, unknown, or even zeros that shouldn't be zero. Your first job is to find all of them.

⚠️ Missing Values in Disguise — Real Industry Examples

Healthcare: Age = 0, Blood pressure = 999 (sentinel values used by old systems)

Banking: Income = -1, Credit score = 9999 (system-level "not available" codes)

E-commerce: Rating = "N/A", delivery_days = 0 (item not yet delivered)

Surveys: Response = "Prefer not to say", "Don't know", empty string ""

Sensor data: Temperature = -999.9 (sensor offline / not connected)

Step-by-Step Detection Process

1

Basic null check — pandas .isnull() and .info()

The first thing you run on any new dataset. This catches NaN, None, and NaT (missing dates).

python
import pandas as pd
import numpy as np

# Load dataset
df = pd.read_csv('customer_data.csv')

# 1. Quick overview — see dtypes + non-null count per column
df.info()

# 2. Count missing per column
df.isnull().sum()

# 3. Percentage of missing per column
missing_pct = (df.isnull().sum() / len(df)) * 100
print(missing_pct.sort_values(ascending=False).round(2))

# 4. Total missing cells in entire dataset
print(f"Total missing: {df.isnull().sum().sum()} / {df.size} cells")
print(f"Missing %: {df.isnull().mean().mean()*100:.2f}%")
2

Hunt for disguised missing values — sentinel/placeholder values

Real-world data often uses specific numbers or strings to represent "missing". You must know your domain to catch these. After finding them, replace with np.nan.

python
# Replace known sentinel values with NaN
df.replace({
    -1: np.nan,        # Banking: "not applicable"
    999: np.nan,       # Medical: "not recorded"
    9999: np.nan,      # Legacy system codes
    '': np.nan,        # Empty strings
    'N/A': np.nan,    # String "N/A"
    'n/a': np.nan,
    'NA': np.nan,
    'null': np.nan,
    'unknown': np.nan,
    'Unknown': np.nan,
    '?': np.nan,       # UCI ML Repository datasets use this!
}, inplace=True)

# Check for suspicious zeros (domain-specific)
# Age=0, Height=0, or Income=0 are biologically/logically impossible
suspicious_cols = ['age', 'income', 'height', 'weight']
for col in suspicious_cols:
    if col in df.columns:
        zero_count = (df[col] == 0).sum()
        print(f"{col}: {zero_count} zeros — check if valid!")
3

Visualize missing patterns — heatmap and bar chart

Numbers tell you HOW MUCH is missing. Visualization tells you WHERE and in what pattern — this is critical for diagnosing the type of missingness (MCAR/MAR/MNAR, explained next).

python
import missingno as msno
import matplotlib.pyplot as plt
import seaborn as sns

# 1. Matrix view — see WHERE nulls appear row by row
# White = missing, dark = present. Patterns reveal type of missingness!
msno.matrix(df, figsize=(12, 6))
plt.title("Missing Value Matrix — White = Missing")
plt.show()

# 2. Bar chart — missing count per column
msno.bar(df, figsize=(12, 4))
plt.show()

# 3. Heatmap — which columns go missing TOGETHER?
# High correlation = missing at same time = possible MAR/MNAR
msno.heatmap(df, figsize=(10, 8))
plt.show()

# 4. Dendogram — cluster columns by missing patterns
msno.dendrogram(df)
plt.show()
4

Check missing patterns by group — cross-tabulation

Check if missing values are concentrated in a specific group (e.g., missing income only for women, or missing test results only for elderly patients). This reveals MAR and MNAR patterns.

python
# Is 'income' missing more for one gender group?
df.groupby('gender')['income'].apply(lambda x: x.isnull().mean() * 100)

# Correlation between 'is_income_missing' indicator and other features
df['income_missing'] = df['income'].isnull().astype(int)
df.corr()['income_missing'].sort_values(ascending=False)

# t-test: is the OBSERVED income different from MISSING income group?
from scipy import stats
obs    = df[df['income'].notna()]['age']
miss   = df[df['income'].isnull()]['age']
t, p   = stats.ttest_ind(obs, miss)
print(f"p-value: {p:.4f}")
# p < 0.05 → age is different in missing group → likely MAR or MNAR
Industry Checklist — Before Any Treatment

① Run df.info() and df.isnull().sum() — get the count

② Replace known sentinel values (-1, 999, "N/A") with np.nan

③ Check suspicious zeros (age, height, income) — are they valid?

④ Visualize with missingno.matrix() to see patterns

⑤ Run cross-tabulation by groups to detect MAR/MNAR signals

⑥ Calculate % missing per column — under 5%? 5–30%? Over 30%? This drives your strategy.

Section 02
Types of Missing Data
MCAR · MAR · MNAR — the most important classification in missing data theory

Before choosing a treatment technique, you must understand WHY the data is missing. This is the single most important diagnostic step. The type of missingness determines which methods are valid and which will introduce bias.

MCAR
Missing Completely At Random

The probability of a value being missing has absolutely nothing to do with any variable — observed or unobserved. It's purely random, like a coin flip.

Test: No statistically significant difference between missing and non-missing groups across any variable.

📋 Real example: A lab technician accidentally spills coffee on some paper records. Which records are destroyed is purely random — has nothing to do with patient age, condition, or severity.
📱 Survey example: A random 10% of survey forms were corrupted during digital scanning — not related to any answer.
MAR
Missing At Random

The probability of missing is related to OTHER observed variables, but NOT to the missing value itself. Once you condition on observed data, the missingness is random.

Test: Statistically significant difference in other columns between missing and non-missing groups.

💰 Real example: Older patients are less likely to fill in online income surveys (because they're less tech-savvy). Income is missing, but the reason (age) is fully observed — not because of their income amount itself.
🏥 Healthcare: Men are less likely to report depression symptoms. Gender is observed, so conditional on gender, the missingness is random.
MNAR
Missing Not At Random

The probability of missing is directly related to the value that's missing itself. This is the hardest to handle and most dangerous to ignore.

Test: No observed variable can explain the pattern — the missing value itself causes the missingness.

💸 Real example: Very high earners and very low earners both refuse to report their income — for opposite reasons (privacy vs. shame). The missingness is caused by the income itself.
🩺 Healthcare: Patients with severe depression skip the depression questionnaire — so the most severe cases are missing from the depression column.

🚨 The Most Dangerous Mistake in Industry

Most data scientists assume MCAR and apply simple imputation — without ever checking the type. If the data is actually MNAR (e.g., sick patients missing their severity scores), then mean imputation will systematically underestimate severity, making your medical model think patients are healthier than they are.

Always diagnose before treating. Never assume MCAR in production.

Section 03
Should You Treat Missing Values at All?
sometimes missing values carry signal — don't blindly fill them

Not every missing value needs to be filled. In some cases, the fact that a value is missing is itself highly informative. Blindly imputing it destroys that signal.

❌ When NOT to Treat (Leave Missing or Flag)

MNAR data: If people with the highest income don't report it, imputing the mean income will corrupt your model badly.

Missingness is the signal: In credit scoring, missing payment history can itself mean "never had credit" — this IS the feature.

Domain says impossible: A hospital visit column missing means the patient never visited — don't impute a visit count.

Feature engineering opportunity: Create a binary flag (is_income_missing = 1/0) and keep the column. Both carry information.

✅ When to Treat Missing Values

Model requires no nulls: Linear regression, SVM, Neural Networks crash on NaN. You must impute.

MCAR with small %: Less than 5% missing randomly → simple deletion or mean imputation is safe.

MAR with observed correlates: Use conditional imputation (group mean, regression) to fill correctly.

Reporting/dashboards: Stakeholders need complete tables — impute for display, but track missing separately.

📏 Industry Rules of Thumb — Missing % Thresholds

< 5% missing  →  Any technique works. Deletion is fine. Simple imputation safe. Impact is minimal.

5% – 20% missing  →  Use smart imputation (group mean, KNN, regression). Avoid deletion — you'll lose too much data.

20% – 40% missing  →  Use advanced methods (MICE, model-based). Always add a missing indicator flag alongside.

> 40% missing  →  Seriously consider dropping the column entirely. Any imputation at this level introduces more noise than signal. Consult domain expert first.

💡 The Indicator Flag Trick — Best of Both Worlds

Instead of choosing between "impute" or "drop", do both simultaneously:

① Keep original column: impute missing values with median (so model can train)

② Add new column column_was_missing (1 = was missing, 0 = was present)

This way, the model learns from the imputed values AND learns that missingness itself is a signal. This is the gold standard approach for MNAR data in production.

Category 1 — Deletion Methods
remove rows or columns with missing data · simplest approach
🗑️
Technique 1.1
Listwise Deletion (Complete Case Analysis) Easy

What it does: Remove any row that has at least one missing value. Only keep rows where ALL columns have values.

When to use: Data is MCAR. Missing % is very small (<5%). You have plenty of rows to spare. Model cannot handle NaN.

When NOT to use: MAR or MNAR data. High missing %. When deleted rows are systematically different from kept rows.

#AgeIncomeScoreResult
12850000720✔ Kept
234NaN680✘ Deleted
3NaN72000750✘ Deleted
44595000800✔ Kept
python
# Drop ALL rows with any NaN
df_clean = df.dropna()

# Drop rows only if NaN in specific important columns
df_clean = df.dropna(subset=['income', 'age'])

# Drop rows where at least 2 columns are missing
df_clean = df.dropna(thresh=df.shape[1] - 1)  # allow max 1 missing
⚠️ Industry Warning

If 30% of rows have any missing value and you drop all of them, you lose 30% of your data. Worse, if those 30% are systematically different (e.g., poorer customers rarely fill income), your model will be biased toward wealthier customers.

📋
Technique 1.2
Column Deletion Easy

What it does: Drop an entire column (feature) if it has too many missing values to be useful.

When to use: A column has >40–60% missing. The column is not important for the model. No good imputation method exists for this feature.

When NOT to use: The column is a key feature even with many missing values. You can engineer a "was_missing" flag to preserve signal.

python
# Drop columns with more than 40% missing
threshold = 0.4
df_clean = df.dropna(axis=1, thresh=int(threshold * len(df)))

# Or manually identify and drop
cols_to_drop = df.columns[df.isnull().mean() > 0.4]
print(f"Dropping: {list(cols_to_drop)}")
df = df.drop(columns=cols_to_drop)
Section 05
Category 2 — Simple (Statistical) Imputation
fill missing values with a single statistic · fast and widely used
📊
Technique 2.1
Mean Imputation Easy

What it does: Fill missing values with the column's average (mean).

Best for: Numerical data. MCAR missingness. When distribution is roughly normal (no extreme outliers).

Avoid when: Data is skewed (outliers pull mean away from center). Data is MNAR. You have outliers in the column.

Salary (Before)Salary (After Mean Imputation)
50,00050,000
NaN71,000 ← filled with mean
80,00080,000
84,00084,000
NaN71,000 ← filled with mean
⚠️ Problem: Reduces Variance

Mean imputation makes all missing rows look "average". This artificially shrinks the variance of the column, which can mislead models that depend on variance (like PCA, linear regression). For a small number of missing values, this is acceptable. For many missing values, it distorts the data.

python
from sklearn.impute import SimpleImputer

# Mean imputation
imp = SimpleImputer(strategy='mean')
df[['salary', 'age']] = imp.fit_transform(df[['salary', 'age']])

# Or with pandas directly
df['salary'].fillna(df['salary'].mean(), inplace=True)
📈
Technique 2.2
Median Imputation Easy

What it does: Fill missing values with the column's median (middle value when sorted).

Best for: Numerical data with skewed distribution or outliers. Income, house prices, age — real-world data is often skewed.

Why better than mean in many cases: Outliers don't affect the median. If your salary column has a few millionaires (1M, 2M, 5M), they'll pull the mean way up but won't affect the median at all.

✅ Industry Standard for Numerical Data

In most production pipelines, median is preferred over mean for numerical imputation because real-world data is almost always right-skewed. Income, transaction amounts, property values, time durations — all are skewed. Median is robust to this.

python
imp = SimpleImputer(strategy='median')
df[['income']] = imp.fit_transform(df[['income']])

# Pandas shortcut
df['income'].fillna(df['income'].median(), inplace=True)
🏷️
Technique 2.3
Mode Imputation Easy

What it does: Fill missing values with the most frequently occurring value (mode).

Best for: Categorical data. Ordinal data. Binary features. Low cardinality columns (few unique values).

Avoid when: No clear dominant category (all categories equally frequent). Very high cardinality (100+ unique categories — mode is meaningless).

python
imp = SimpleImputer(strategy='most_frequent')
df[['city', 'gender']] = imp.fit_transform(df[['city', 'gender']])

# Pandas shortcut
df['city'].fillna(df['city'].mode()[0], inplace=True)
🔢
Technique 2.4
Constant / Arbitrary Value Imputation Easy

What it does: Fill all missing values with a specific constant you define — 0, -1, "Missing", "Unknown".

Best for: When missing truly means zero (e.g., no purchase = 0 purchases). Categorical columns where "Unknown" is a valid category that the model should learn from. Tree-based models (XGBoost, Random Forest) that can learn from "Missing" as a category.

✅ Good Use Case

E-commerce: num_prev_purchases = NaN → fill with 0. This is logically correct — a new customer has 0 previous purchases.

Categorical: city = NaN → fill with "Unknown" and let the model learn from it.

❌ Bad Use Case

Age = 0: Filling a missing age with 0 creates a biologically impossible record that will confuse the model badly.

Income = -1: If -1 was used as a sentinel to begin with, filling with -1 just recreates the problem.

python
# Fill with 0 (purchases, counts)
df['num_purchases'].fillna(0, inplace=True)

# Fill categorical with explicit "Unknown" label
df['city'].fillna('Unknown', inplace=True)

# sklearn way with constant
imp = SimpleImputer(strategy='constant', fill_value='Missing')
df[['product_category']] = imp.fit_transform(df[['product_category']])
👥
Technique 2.5
Group-Based (Conditional) Mean/Median Imputation Medium

What it does: Instead of filling with the overall mean, fill with the mean of the relevant group that the row belongs to.

Best for: MAR data (where missingness is related to another observed variable). Any time you know that subgroups have very different distributions.

Real example: Instead of filling missing income with the overall mean (₹55,000), fill it with the mean income for that person's specific profession — doctors get doctor average, teachers get teacher average. Far more accurate.

PersonProfessionIncomeAfter Group Imputation
1DoctorNaN₹1,20,000 ← doctors' median
2TeacherNaN₹45,000 ← teachers' median
3Doctor₹1,10,000₹1,10,000
4Teacher₹42,000₹42,000
python
# Fill missing income with the median income of the same profession group
df['income'] = df.groupby('profession')['income'].transform(
    lambda x: x.fillna(x.median())
)

# Multiple groupby levels (profession + city)
df['income'] = df.groupby(['profession', 'city'])['income'].transform(
    lambda x: x.fillna(x.median())
)

# If subgroup itself is all NaN, fallback to global median
global_median = df['income'].median()
df['income'].fillna(global_median, inplace=True)
✅ Industry Favorite for MAR Data

This is the most used "simple" technique in real projects. Groupby + transform is fast, intuitive, and much smarter than global mean/median. Used heavily in banking (impute by customer segment), healthcare (impute by age+gender group), and retail (impute by product category).

Section 06
Category 3 — Advanced Imputation
KNN · Interpolation · MICE · use relationships between columns
🔍
Technique 3.1
KNN Imputation (K-Nearest Neighbors) Medium

What it does: For each missing value, finds the K most similar rows (neighbors) based on all other features, then fills with their average (numerical) or most common value (categorical).

Why it's smart: It uses ALL the other available information about that row to find truly similar records. A 25-year-old software engineer's missing income is filled using other 25-year-old software engineers — not the global average.

Best for: Medium-sized datasets. When features are correlated. MAR data. When you want accuracy over speed.

Avoid when: Very large datasets (slow — calculates distance to every row). High-dimensional data (curse of dimensionality). When computation time matters.

// How KNN Imputation works, step by step: Row 5: Age=28, Profession=Engineer, City=Mumbai, Income=NaN Step 1: Find K=5 rows most similar to Row 5 (based on Age, Profession, City) Similar rows' incomes: [62000, 70000, 68000, 65000, 72000] Step 2: Fill missing income = mean of neighbors = (62000 + 70000 + 68000 + 65000 + 72000) / 5 = 67,400 ← much smarter than global mean of 55,000
python
from sklearn.impute import KNNImputer

# n_neighbors = K. Start with 5, tune with cross-validation
knn_imp = KNNImputer(n_neighbors=5, weights='uniform')
df_imputed = knn_imp.fit_transform(df_numeric)  # numeric columns only

# Important: scale features before KNN (distance-sensitive)
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df_numeric)
df_imputed = knn_imp.fit_transform(df_scaled)
df_imputed = scaler.inverse_transform(df_imputed)  # back to original scale
📉
Technique 3.2
Interpolation Medium

What it does: Fill missing values by estimating from neighboring values in a sequence. Assumes values change smoothly between known points.

Best for: Time-series data. Sequential data. Stock prices, temperature readings, sensor data, website traffic over time.

Avoid when: Data is not sequential/ordered. Missing values appear at the beginning or end of the series (extrapolation is dangerous).

python
# Sort by time first (critical!)
df = df.sort_values('date').set_index('date')

# Linear interpolation (good for steadily changing values)
df['temperature'] = df['temperature'].interpolate(method='linear')

# Time-based interpolation (accounts for irregular time gaps)
df['stock_price'] = df['stock_price'].interpolate(method='time')

# Spline interpolation (smooth curves — good for sensor data)
df['sensor'] = df['sensor'].interpolate(method='spline', order=3)

# Forward fill: use last known value (stock prices, IoT)
df['price'] = df['price'].fillna(method='ffill')

# Backward fill: use next known value
df['price'] = df['price'].fillna(method='bfill')

Technique 3.3
MICE — Multiple Imputation by Chained Equations Advanced

What it does: The most statistically rigorous imputation method. It imputes each missing column by building a separate regression model that uses all other columns as predictors. It runs multiple rounds (iterations) until convergence, producing multiple complete datasets and combining them.

How it works (simplified):

// MICE process — iterative imputation Round 1: Fill all missing values with column medians (initial estimate) Round 2: For each column with missing values: → Treat that column as the TARGET (y) → Use all other columns as FEATURES (X) → Train a regression model on observed rows → Predict and fill the missing rows → Move to next column Round 3: Repeat Round 2 (now with better estimates in other columns) Round 4, 5 ... until values stop changing significantly Result: Each missing value gets a statistically grounded prediction that uses ALL available information in the dataset

Best for: High % missing (20-40%). MAR data. When accuracy matters more than speed. Research, healthcare, finance — any high-stakes application.

Avoid when: Large datasets (slow). When you need a simple, explainable pipeline. When columns are not actually correlated.

python
from sklearn.experimental import enable_iterative_imputer
from sklearn.impute import IterativeImputer
from sklearn.ensemble import RandomForestRegressor

# Basic MICE (uses BayesianRidge regression internally by default)
mice_imp = IterativeImputer(
    max_iter=10,          # number of rounds
    random_state=42,
    tol=1e-3              # stop when values change < this
)
df_imputed = mice_imp.fit_transform(df_numeric)

# Advanced: use Random Forest as the estimator (more accurate)
rf_imp = IterativeImputer(
    estimator=RandomForestRegressor(n_estimators=100, random_state=42),
    max_iter=10,
    random_state=42
)
df_imputed = rf_imp.fit_transform(df_numeric)
✅ Industry Gold Standard for Research & High-Stakes ML

MICE is used in medical research (clinical trials), economic analysis, and anywhere the imputed values directly affect life/money decisions. It's the statistically correct method when data is MAR with 20%+ missing.

Section 07
Category 4 — Model-Based Imputation
train a separate model just to predict missing values
🌳
Technique 4.1
Random Forest / Tree-Based Imputation Advanced

What it does: Train a Random Forest model where the column with missing values is the target variable, and all other (complete) columns are features. Predict the missing values using this model.

Why it's powerful: Handles non-linear relationships, doesn't need feature scaling, works with both numerical and categorical features. Automatically captures complex interactions between columns.

Best for: Complex, non-linear relationships between features. Large datasets. When you have many complete columns to use as predictors.

python
from sklearn.ensemble import RandomForestRegressor

# Split dataset into rows where 'income' is known vs missing
known   = df[df['income'].notna()]
missing = df[df['income'].isnull()]

features = ['age', 'education', 'profession', 'city']

X_train = known[features]
y_train = known['income']
X_pred  = missing[features]

# Train RF to predict income from other features
rf = RandomForestRegressor(n_estimators=100, random_state=42)
rf.fit(X_train, y_train)

# Fill in the missing values with predictions
df.loc[df['income'].isnull(), 'income'] = rf.predict(X_pred)
📐
Technique 4.2
Regression Imputation Medium

What it does: Build a linear (or logistic for categorical) regression model to predict the missing column from other columns. Fill missing values with the regression predictions.

Best for: When a clear linear relationship exists between the missing column and predictors. Fast and explainable. Good for MAR data.

Limitation: Assumes linear relationships. Adds no randomness (all predictions come from the same regression line — reduces variance). For randomness, use Stochastic Regression: add a random residual to each prediction.

python
from sklearn.linear_model import LinearRegression
import numpy as np

known   = df[df['income'].notna()]
missing = df[df['income'].isnull()]
features = ['age', 'years_experience']

lr = LinearRegression()
lr.fit(known[features], known['income'])

preds = lr.predict(missing[features])

# Stochastic: add random noise from residuals (preserves variance)
residuals = known['income'] - lr.predict(known[features])
noise = np.random.choice(residuals, size=len(preds))
df.loc[df['income'].isnull(), 'income'] = preds + noise

Category 5 — Missing Indicator Flag Method
preserve missingness as information · use alongside imputation
🚩
Technique 5.1
Missing Indicator (Binary Flag) Easy

What it does: Create a new binary column [column_name]_was_missing (1 = was missing, 0 = was present). Then impute the original column separately. Keep both in the dataset.

Why this is powerful: Missingness itself can be a feature. In credit scoring, if someone never answered the income question, that pattern is different from someone who provided their income. The model learns from both the filled value AND the fact that it was filled.

Best for: MNAR data. Any column where missingness has potential meaning. Always use when % missing is significant (>10%).

BeforeIncomeAfter ImputationIncomeincome_was_missing
Row 1₹50,000→₹50,0000
Row 2NaN→₹62,0001
Row 3NaN→₹62,0001
Row 4₹90,000→₹90,0000
python
# Manual approach — create flag column before imputing
cols_with_missing = ['income', 'credit_score', 'age']

for col in cols_with_missing:
    df[f'{col}_was_missing'] = df[col].isnull().astype(int)

# Now impute the original columns
df[cols_with_missing] = df[cols_with_missing].fillna(df[cols_with_missing].median())

# sklearn's MissingIndicator (works with pipelines)
from sklearn.impute import MissingIndicator
indicator = MissingIndicator(features='all')
missing_flags = indicator.fit_transform(df)

# Add flags to original dataframe
flag_cols = [f'{col}_flag' for col in df.columns]
df_flags = pd.DataFrame(missing_flags, columns=flag_cols)
df_final = pd.concat([df, df_flags], axis=1)
✅ Always Add Flags for MNAR Data

This technique is used in virtually every production ML pipeline at companies like Google, Amazon, and Netflix. It costs almost nothing (just an extra binary column) and can significantly improve model performance by letting it learn that "this person didn't answer" is itself a pattern.

🌳
Technique 5.2
Let Tree-Based Models Handle NaN Natively Easy

What it does: Some models can natively handle missing values — you don't need to impute at all.

XGBoost, LightGBM, CatBoost: These tree-based boosting models learn the best direction to send missing values at each split. They can handle NaN in training AND test data without any preprocessing.

Best for: Tabular data competitions. Production ML on tabular data. When you want zero imputation overhead.

Not applicable to: Linear regression, SVM, Neural Networks, Logistic Regression — these crash on NaN and require explicit imputation.

python
import xgboost as xgb
import lightgbm as lgb

# XGBoost handles NaN natively — no imputation needed!
model = xgb.XGBClassifier(random_state=42)
model.fit(X_train, y_train)  # X_train can contain NaN values
preds = model.predict(X_test)

# LightGBM also handles NaN natively
lgb_model = lgb.LGBMClassifier(random_state=42)
lgb_model.fit(X_train, y_train)  # same — NaN is fine

# CatBoost handles categorical NaN too
from catboost import CatBoostClassifier
cat_model = CatBoostClassifier(verbose=0)
cat_model.fit(X_train, y_train, cat_features=['city', 'gender'])

Decision Framework

Use this decision tree (yes, ironic!) to navigate missing value treatment in any real project:

import pandas as pd
import numpy as np
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer, KNNImputer, MissingIndicator
from sklearn.experimental import enable_iterative_imputer
from sklearn.impute import IterativeImputer
from sklearn.preprocessing import StandardScaler, OrdinalEncoder
import missingno as msno

# ─────────────────────────────────────────────────────
# STEP 1: DIAGNOSE — Always diagnose before treating
# ─────────────────────────────────────────────────────
def diagnose_missing(df):
    """Comprehensive missing value diagnosis report"""
    missing = df.isnull().sum()
    pct     = (missing / len(df) * 100).round(2)
    dtype   = df.dtypes

    report = pd.DataFrame({
        'missing_count': missing,
        'missing_pct':   pct,
        'dtype':         dtype
    }).query('missing_count > 0').sort_values('missing_pct', ascending=False)

    print("="*55)
    print("MISSING VALUE DIAGNOSIS REPORT")
    print("="*55)
    print(report.to_string())
    print(f"\nTotal rows: {len(df)}")
    print(f"Total missing cells: {missing.sum()}")
    print(f"Overall missing %: {(missing.sum()/df.size*100):.2f}%")
    return report

report = diagnose_missing(df)

# ─────────────────────────────────────────────────────
# STEP 2: CLEAN — Replace sentinel values first
# ─────────────────────────────────────────────────────
df.replace([-1, 999, 9999, '', 'N/A', '?', 'unknown'], np.nan, inplace=True)

# ─────────────────────────────────────────────────────
# STEP 3: DROP high-missing columns (>40%)
# ─────────────────────────────────────────────────────
high_miss_cols = report[report['missing_pct'] > 40].index.tolist()
print(f"Dropping columns with >40% missing: {high_miss_cols}")
df = df.drop(columns=high_miss_cols)

# ─────────────────────────────────────────────────────
# STEP 4: ADD MISSING INDICATOR FLAGS for MNAR columns
# ─────────────────────────────────────────────────────
# These are columns where missingness itself carries information
mnar_columns = ['income', 'credit_score']

for col in mnar_columns:
    if col in df.columns:
        df[f'{col}_was_missing'] = df[col].isnull().astype(int)

# ─────────────────────────────────────────────────────
# STEP 5: GROUP-BASED IMPUTATION for MAR columns
# ─────────────────────────────────────────────────────
# income is MAR — related to profession
df['income'] = df.groupby('profession')['income'].transform(
    lambda x: x.fillna(x.median())
)
df['income'].fillna(df['income'].median(), inplace=True)  # fallback

# ─────────────────────────────────────────────────────
# STEP 6: SKLEARN PIPELINE for remaining columns
# ─────────────────────────────────────────────────────
num_features = ['age', 'credit_score', 'loan_amount']
cat_features = ['city', 'gender', 'education']

# Numerical: median imputation + scaling
num_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler',  StandardScaler()),
])

# Categorical: mode imputation + encoding
cat_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OrdinalEncoder(handle_unknown='use_encoded_value', unknown_value=-1)),
])

# Combine both into one ColumnTransformer
preprocessor = ColumnTransformer([
    ('num', num_pipeline, num_features),
    ('cat', cat_pipeline, cat_features),
])

# Final pipeline with a model at the end
from sklearn.ensemble import RandomForestClassifier
full_pipeline = Pipeline([
    ('preprocessing', preprocessor),
    ('model', RandomForestClassifier(random_state=42)),
])

💡 Production Best Practices

Always fit imputers on training data only. Never on the full dataset. Then transform both train and test. This prevents data leakage.

Store the fitted pipeline. Save it with joblib.dump(full_pipeline, 'pipeline.pkl') so the same imputation strategy applies to new incoming data.

Track imputation in MLflow/DVC. Log which columns were imputed, with what strategy, and what the pre/post distributions looked like. Helps debug model drift.

Validate after imputation. Check that distributions haven't shifted dramatically. Plot histograms before vs. after.

Complete Summary — All Techniques

Everything you need to handle missing values like a data scientist at a top tech company.

1. Identify

df.isnull().sum(), missingno.matrix(), check sentinel values (-1, 999, "N/A"), t-test by groups

2. Diagnose Type

MCAR: random, no pattern. MAR: explained by other columns. MNAR: caused by the missing value itself — most dangerous.

3. Deletion

Listwise: remove rows. Column drop: remove features with >40-60% missing. Only safe for MCAR with small %.

4. Simple Imputation

Mean (normal data), Median (skewed data ⭐), Mode (categorical ⭐), Constant, Group-Based Median (MAR ⭐)

5. Advanced Imputation

KNN: uses similar rows. Interpolation: time-series (⭐). MICE: iterative regression, gold standard for 20-40% MAR.

6. Model-Based

Random Forest imputation: train RF to predict missing column. Regression imputation: linear relationship assumption.

7. Indicator Flag

Always add [col]_was_missing=1/0 for MNAR data. Let model learn from missingness itself. Free performance gain.

8. Native NaN

XGBoost, LightGBM, CatBoost handle NaN natively. No imputation needed. Best for tabular ML in production.
