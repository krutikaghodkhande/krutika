What is Logistic Regression?

Logistic Regression is used for binary classification problems where we predict probability of an event occurring (0 or 1, Yes/No, True/False).

Key Differences from Linear Regression:

Linear Regression

  • Predicts continuous values
  • Output: Any real number
  • Example: House prices (₹10L, ₹15L)
  • Line: Straight line

Logistic Regression

  • Predicts probabilities (0 to 1)
  • Output: Between 0 and 1
  • Example: Spam (0.8), Not Spam (0.2)
  • Curve: S-shaped curve
Logistic Regression Equation:

p = 1 / (1 + e^-(β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ))

Where:
• p = Probability (0 to 1)
• e = Euler's number (≈ 2.718)
• β₀, β₁, β₂... = Coefficients to be learned

🎯 Real Example - Email Spam Detection:

Problem: Predict if email is spam based on number of suspicious words

Input: Email with 5 suspicious words

Output: Probability = 0.85 (85% chance it's spam)

Decision: If p > 0.5 → Spam, else → Not Spam

Sigmoid Function - The Heart of Logistic Regression

The Sigmoid Function transforms any real number into a value between 0 and 1, making it perfect for probabilities.

Sigmoid Formula:

σ(z) = 1 / (1 + e^(-z))

Where: z = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ

📊 Sigmoid Function Properties:

Key Properties of Sigmoid:

  • Range: Always between 0 and 1
  • S-Shape: Smooth curve, steep in middle
  • At z=0: σ(0) = 0.5 (50% probability)
  • Large positive z: σ(z) approaches 1
  • Large negative z: σ(z) approaches 0

🔢 Step-by-step Example:

Student Pass/Fail Prediction based on study hours:
Model: p = 1/(1 + e^(-(-5 + 1×study_hours)))

For 3 hours: z = -5 + 1×3 = -2
p = 1/(1 + e^2) = 1/(1 + 7.39) = 0.12 (12% chance of passing)

For 8 hours: z = -5 + 1×8 = 3
p = 1/(1 + e^(-3)) = 1/(1 + 0.05) = 0.95 (95% chance of passing)

🎯 Why Sigmoid?

  • Range [0,1]: Perfect for probabilities
  • Smooth curve: Differentiable everywhere
  • S-shape: Gradual transition from 0 to 1
  • Interpretable: Clear probability meaning

Cost Function in Logistic Regression

We cannot use MSE (Mean Squared Error) for logistic regression because it creates a non-convex function with multiple local minima.

❌ Why Not MSE?

Problem: MSE + Sigmoid = Non-convex function

  • Multiple local minima (valleys)
  • Gradient descent might get stuck
  • No guarantee of finding global minimum

✅ Solution: Log-Likelihood Cost Function

Logistic Regression Cost Function:

J(θ) = -(1/m) × Σ[yᵢ×log(hᵢ) + (1-yᵢ)×log(1-hᵢ)]

Where:
• m = Number of training examples
• yᵢ = Actual label (0 or 1)
• hᵢ = Predicted probability
• log = Natural logarithm

🔢 Example Calculation:

Scenario: Email spam detection with 3 emails

EmailActual (y)Predicted (h)Cost Calculation
11 (Spam)0.8-log(0.8) = 0.22
20 (Not Spam)0.3-log(1-0.3) = 0.36
31 (Spam)0.9-log(0.9) = 0.11

Total Cost: (0.22 + 0.36 + 0.11) / 3 = 0.23

🎯 Cost Function Intuition:

  • Correct prediction: Low cost (reward good predictions)
  • Wrong prediction: High cost (penalize bad predictions)
  • Very wrong prediction: Infinite cost (heavily penalize)

Log Loss - Detailed Explanation

Log Loss (Logarithmic Loss) is the cost function used in logistic regression. It measures the uncertainty of predictions.

Log Loss Formula (for single prediction):

Loss = -[y×log(p) + (1-y)×log(1-p)]

Where:
• y = Actual label (0 or 1)
• p = Predicted probability
• This gives us two cases:

When y = 1 (Positive Class):

Loss = -log(p)

  • If p = 0.9 → Loss = -log(0.9) = 0.11
  • If p = 0.5 → Loss = -log(0.5) = 0.69
  • If p = 0.1 → Loss = -log(0.1) = 2.30

When y = 0 (Negative Class):

Loss = -log(1-p)

  • If p = 0.1 → Loss = -log(0.9) = 0.11
  • If p = 0.5 → Loss = -log(0.5) = 0.69
  • If p = 0.9 → Loss = -log(0.1) = 2.30

📊 Log Loss Behavior:

Key Insights about Log Loss:

  • Perfect prediction: Loss = 0
  • Confident wrong prediction: Loss → ∞
  • Uncertain prediction (p=0.5): Loss = 0.69
  • Exponential penalty: Cost increases rapidly for wrong predictions

🎯 Real Example - Medical Diagnosis:

Disease Detection Model:

Patient 1: Actually has disease (y=1), Model predicts 0.95
Loss = -log(0.95) = 0.05 (Very low - good prediction)

Patient 2: Actually has disease (y=1), Model predicts 0.2
Loss = -log(0.2) = 1.61 (High - bad prediction, dangerous miss)

Why this matters: Missing a disease is heavily penalized!

Convex vs Non-convex Functions

Understanding the shape of cost functions is crucial for optimization. Convex functions guarantee global minimum, while non-convex functions have multiple local minima.

✅ Convex Function

Bowl-shaped, Single global minimum

❌ Non-convex Function

Multiple valleys, Many local minima

AspectConvex FunctionNon-convex Function
ShapeBowl-shapedMultiple hills & valleys
MinimaSingle global minimumMultiple local minima
OptimizationAlways finds global minimumMay get stuck in local minimum
Gradient DescentGuaranteed convergenceMay converge to local minimum
ExampleLog Loss in Logistic RegressionMSE with Sigmoid

🔍 Why Log Loss is Convex:

Mathematical Property: Log Loss + Sigmoid = Convex Function

  • Negative log-likelihood is convex
  • Sigmoid function maintains convexity
  • Gradient descent finds the global minimum
  • No worry about getting stuck in local minima

🎯 Practical Impact:

  • Reliable Training: Convex cost ensures consistent results
  • No Random Restarts: Don't need multiple initial points
  • Predictable Convergence: Algorithm always improves

Performance Metrics for Classification

Unlike regression, classification requires different metrics to evaluate model performance. We need to measure how well we classify different categories.

🎯 Why Different Metrics?

  • Binary Outcomes: Right or Wrong (not how much off)
  • Class Imbalance: What if 95% emails are not spam?
  • Different Costs: Missing cancer vs false alarm
  • Business Context: Precision vs Recall trade-offs

  • Precision Also Important: Don't cause unnecessary panic
  • Accuracy Alone Not Enough: 95% healthy people doesn't mean 95% accuracy is good
  • Cost of Errors: Missing cancer >> False alarm

Accuracy - Overall Correctness

Accuracy measures the percentage of correct predictions out of all predictions made.

Accuracy Formula:

Accuracy = (TP + TN) / (TP + TN + FP + FN)

= Correct Predictions / Total Predictions

Range: 0 to 1 (0% to 100%)

📧 Email Spam Example:

Given: TP=25, TN=65, FP=5, FN=5

Calculation:
Accuracy = (25 + 65) / (25 + 65 + 5 + 5)
Accuracy = 90 / 100 = 0.9 = 90%

Interpretation: Model correctly classifies 90% of emails

📊 Accuracy Visualization:

✅ When Accuracy is Good:

  • Balanced dataset (equal classes)
  • All errors have equal cost
  • Simple, easy to understand
  • Good for initial assessment

❌ When Accuracy Misleads:

  • Imbalanced dataset (95% one class)
  • Different error costs
  • Medical diagnosis scenarios
  • Fraud detection problems

⚠️ Accuracy Paradox Example:

Cancer Detection: If only 1% of population has cancer, a model that always predicts "No Cancer" will have 99% accuracy but is completely useless!

Precision - Quality of Positive Predictions

Precision measures how many of our positive predictions were actually correct. It focuses on the quality of positive predictions.

Precision Formula:

Precision = TP / (TP + FP)

= True Positives / All Predicted Positives

Range: 0 to 1 (0% to 100%)

📧 Email Spam Example:

Given: TP=25, FP=5

Calculation:
Precision = 25 / (25 + 5) = 25 / 30 = 0.83 = 83%

Interpretation: When model says "Spam", it's correct 83% of the time

📊 Precision Visualization:

🎯 When is Precision Important?

High Precision Needed:

  • Email Spam: Don't want good emails in spam
  • Drug Testing: False positive = innocent person punished
  • Fraud Detection: Don't block legitimate transactions

Real World Impact:

  • Customer Trust: False positives annoy customers
  • Operational Cost: Manual review of false positives
  • Legal Issues: Wrongful accusations

💡 Precision Intuition:

"When I say YES, how often am I right?"

High precision = Low false positive rate = Trustworthy positive predictions

Recall - Coverage of Actual Positives

Recall measures how many of the actual positive cases we successfully identified. It focuses on not missing positive cases.

Recall Formula:

Recall = TP / (TP + FN)

= True Positives / All Actually Positive

Range: 0 to 1 (0% to 100%)
Also called: Sensitivity, True Positive Rate

📧 Email Spam Example:

Given: TP=25, FN=5

Calculation:
Recall = 25 / (25 + 5) = 25 / 30 = 0.83 = 83%

Interpretation: Model catches 83% of all spam emails

📊 Recall Visualization:

Out of all actual positives, how many did we find?

Found (True Positives) 83%
Missed (False Negatives) 17%

🎯 When is Recall Critical?

High Recall Essential:

  • Cancer Detection: Can't miss any cancer cases
  • Security Systems: Can't miss threats
  • Quality Control: Can't miss defective products

Cost of Low Recall:

  • Life threatening: Missed diseases
  • Safety issues: Undetected problems
  • Financial loss: Missed opportunities

💡 Recall Intuition:

"Of all the actual YES cases, how many did I find?"

High recall = Low false negative rate = Don't miss important cases

⚖️ Precision vs Recall Trade-off:

Usually there's a trade-off: Increase one, decrease the other. The F1-score helps balance both!

F1-Score - Harmonic Mean of Precision & Recall

F1-Score combines precision and recall into a single metric using harmonic mean, giving equal weight to both.

F1-Score Formula:

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Alternative form:
F1 = 2TP / (2TP + FP + FN)

Range: 0 to 1 (0% to 100%)

📧 Email Spam Example:

Given: Precision = 0.83, Recall = 0.83

Calculation:
F1 = 2 × (0.83 × 0.83) / (0.83 + 0.83)
F1 = 2 × 0.69 / 1.66 = 1.38 / 1.66 = 0.83 = 83%

Interpretation: Balanced performance of 83%

📊 F1-Score vs Arithmetic Mean:

Why Harmonic Mean?
ScenarioPrecisionRecallArithmetic MeanF1-Score (Harmonic)Reality
Balanced80%80%80%80%✅ Good
High Precision Only95%30%62.5%45.7%❌ Missing many

🎯 Why Harmonic Mean?

Arithmetic Mean Problem:

Precision = 100%, Recall = 10%

Average = (100 + 10)/2 = 55%

❌ Misleading! Model is terrible

Harmonic Mean Solution:

Precision = 100%, Recall = 10%

F1 = 2×(100×10)/(100+10) = 18%

✅ Correctly shows poor performance

🎯 When to Use F1-Score:

  • Imbalanced datasets
  • Both precision and recall matter equally
  • Single metric needed for model comparison
  • Binary classification evaluation

ROC Curve & AUC

ROC (Receiver Operating Characteristic) curve shows model performance across all classification thresholds. AUC (Area Under Curve) summarizes ROC into a single number.

ROC Curve Components:

X-axis: False Positive Rate (FPR) = FP / (FP + TN)
Y-axis: True Positive Rate (TPR) = TP / (TP + FN) = Recall

AUC Range: 0 to 1
• AUC = 1: Perfect classifier
• AUC = 0.5: Random classifier
• AUC < 0.5: Worse than random

Regularization in Logistic Regression

Just like linear regression, logistic regression can overfit when we have too many features or complex models. Regularization helps prevent this.

🚫 Overfitting in Logistic Regression:

  • Model memorizes training data instead of learning patterns
  • Very high training accuracy but poor test accuracy
  • Coefficients become very large
  • Decision boundary becomes too complex
Regularized Cost Function:

J(θ) = Original Cost + λ × Penalty Term

L1 (Lasso): J(θ) = -Log Likelihood + λ × Σ|θᵢ|

L2 (Ridge): J(θ) = -Log Likelihood + λ × Σθᵢ²

ElasticNet: J(θ) = -Log Likelihood + λ₁×Σ|θᵢ| + λ₂×Σθᵢ²

L1 Regularization

  • Feature selection
  • Sparse models
  • Some coefficients → 0
  • Interpretable results

L2 Regularization

  • Shrinks coefficients
  • Keeps all features
  • Smooth solutions
  • Handles multicollinearity

ElasticNet

  • Combines L1 + L2
  • Balanced approach
  • Feature selection + shrinkage
  • Best of both worlds

🎯 Medical Diagnosis Example:

Problem: Predict heart disease using 100+ medical features

Without Regularization: Model uses all 100+ features, overfits training data

With L1 Regularization: Model selects only 15 most important features

Result: Better generalization, more interpretable, easier for doctors to use

Summary & Key Takeaways

🎯 Logistic Regression:

  • Purpose: Binary classification
  • Output: Probabilities (0 to 1)
  • Key: Sigmoid function
  • Decision: Threshold (usually 0.5)

🔄 Sigmoid & Cost:

  • Sigmoid: Maps any number to [0,1]
  • Log Loss: Convex cost function
  • Convex: Guaranteed global minimum
  • Optimization: Gradient descent works well

📊 Performance Metrics:

  • Confusion Matrix: Foundation of all metrics
  • Accuracy: Overall correctness
  • Precision: Quality of positives
  • Recall: Coverage of positives
  • F1: Balance of precision & recall
  • ROC-AUC: Threshold-independent

⚖️ Regularization:

  • L1: Feature selection, sparse models
  • L2: Coefficient shrinkage
  • ElasticNet: Best of both worlds
  • Purpose: Prevent overfitting
