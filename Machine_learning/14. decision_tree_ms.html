Section 01
What is a Decision Tree?
intuition before any math

A Decision Tree is a model that learns to make predictions by asking a series of yes/no questions about the data — like a flowchart. Each question splits the data into groups, and at the end of every path is the final prediction.

Think of how a doctor diagnoses a patient — "Do you have fever? → Yes → Do you have a cough? → Yes → Likely flu." That reasoning is exactly a decision tree.

Example: "Should I carry an umbrella?" Is it cloudy? ├── YES ──→ Is it rainy season? │ ├── YES ──→ ✔ Carry umbrella │ └── NO ──→ ✘ Skip it └── NO ──→ ✘ Skip it
Why use it?

Completely interpretable — you can trace every prediction step by step. Great for explaining decisions to non-technical stakeholders.

How it works?

Finds the feature that best separates the data at each step. Repeats recursively on each branch until leaves are pure or stopping rules trigger.

When to use?

Classification and regression tasks. Works well with both categorical and numerical features. Excellent baseline model.

📌 Key Vocabulary — Learn These First

Root Node — The very first question. The most important feature becomes the root.

Internal Node — Any question in the middle of the tree.

Leaf Node — The final answer. No more questions, just a prediction (Yes/No).

Branch — The path taken based on the answer (e.g. "High" or "Low").

Depth — How many questions you answer from root to leaf.

Impurity — How mixed a node is. All-Yes = pure (good). Mixed Yes+No = impure (bad).

⚠️ The Core Problem the Algorithm Solves

With 3 features (Study Hours, Attendance, Sleep), which one should be the first question?

If we ask the wrong question first, the tree becomes deep and unreliable. The algorithm's job is to find the feature that, when used to split the data, produces the most pure children.

Two algorithms measure "purity" differently: ID3 uses Entropy, CART uses Gini Impurity.

Section 03
ID3 Algorithm
Entropy + Information Gain · step-by-step math
Why Entropy?

Entropy measures disorder. All same class → 0 (ordered). 50-50 split → 1 (maximum disorder). We want to reduce entropy.

How it works?

Measure entropy before split → measure weighted entropy after split → difference is Information Gain. Pick the highest.

When to use?

Categorical features only. Academic projects. Learning theory. ID3 is not available in sklearn.

📘 The Entropy Formula

Entropy tells us how mixed a group is. For two classes (Yes/No):

Entropy(S) = −p(Yes) × log₂(p(Yes)) − p(No) × log₂(p(No)) // log₂(1.0) = 0.0 → if all same class, entropy = 0 (pure, perfect) // log₂(0.5) = −1 → if 50-50 split, entropy = 1 (maximum disorder) // Entropy range: 0 (pure) to 1 (completely mixed)
1

Calculate Root Entropy — before any split

The whole dataset has 6 Yes and 4 No. How disordered is it right now?

p(Yes) = 6/10 = 0.6   ,   p(No) = 4/10 = 0.4 Entropy(root) = −(0.6) × log₂(0.6) − (0.4) × log₂(0.4) = −(0.6) × (−0.737) − (0.4) × (−1.322) = 0.442 + 0.529 = 0.971 ← close to 1, dataset is mostly mixed

This 0.971 is our baseline. Any feature we split on should bring this number down. The more it drops, the better the feature.

2

Split on "Study Hours" — what happens to entropy?

High group (rows 1,2,6,8,10) — 5 students: Yes=4, No=1

Low group (rows 3,4,5,7,9) — 5 students: Yes=2, No=3

E(High) = −(4/5)×log₂(4/5) − (1/5)×log₂(1/5) = −(0.80)×(−0.322) − (0.20)×(−2.322) = 0.722 ← mostly Yes, fairly pure E(Low) = −(2/5)×log₂(2/5) − (3/5)×log₂(3/5) = −(0.40)×(−1.322) − (0.60)×(−0.737) = 0.971 ← still mixed // Weighted entropy = (size of branch / total) × entropy of branch Weighted E = (5/10)×0.722 + (5/10)×0.971 = 0.361 + 0.486 = 0.847 Information Gain = 0.971 − 0.847 = 0.124
3

Split on "Attendance" — what happens to entropy?

Good group (rows 1,2,3,7,8) — 5 students: Yes=4, No=1

Poor group (rows 4,5,6,9,10) — 5 students: Yes=2, No=3

E(Good) = −(4/5)×log₂(4/5) − (1/5)×log₂(1/5) = 0.722 E(Poor) = −(2/5)×log₂(2/5) − (3/5)×log₂(3/5) = 0.971 Weighted E = (5/10)×0.722 + (5/10)×0.971 = 0.847 Information Gain = 0.971 − 0.847 = 0.124
4

Split on "Sleep Hours" — what happens to entropy?

Enough group (rows 1,3,4,8,9,10) — 6 students: Yes=4, No=2

Less group (rows 2,5,6,7) — 4 students: Yes=2, No=2

E(Enough) = −(4/6)×log₂(4/6) − (2/6)×log₂(2/6) = −(0.667)×(−0.585) − (0.333)×(−1.585) = 0.918 E(Less) = −(2/4)×log₂(2/4) − (2/4)×log₂(2/4) = −(0.5)×(−1) − (0.5)×(−1) = 1.000 ← perfectly 50-50! worst possible entropy Weighted E = (6/10)×0.918 + (4/10)×1.000 = 0.551 + 0.400 = 0.951 Information Gain = 0.971 − 0.951 = 0.020 ← very low! barely helps

Study Hours and Attendance both tie at IG = 0.124. Either can be the root node — ID3 picks one (often alphabetical or first encountered).

Section 04
CART Algorithm
Gini Impurity · used inside scikit-learn
Why Gini?

No logarithm needed — just squaring. Faster to compute. Asks: "if I randomly pick 2 records, what's the chance they're different classes?"

How it works?

Calculate Gini for each split. Pick the feature with the lowest weighted Gini (most pure children). Always binary splits.

When to use?

In practice — always. sklearn's DecisionTreeClassifier uses CART. Works for regression too.

📘 The Gini Formula

Gini measures how often a randomly chosen element would be incorrectly classified if randomly labeled according to the distribution in the node.

Gini(S) = 1 − p(Yes)² − p(No)² // Gini = 0 → perfectly pure (all same class) ← best // Gini = 0.5 → maximally impure (50-50 split) ← worst // Note: Gini max is 0.5, unlike Entropy which maxes at 1 Gini(root) = 1 − (0.6)² − (0.4)² = 1 − 0.36 − 0.16 = 0.480 ← our baseline
1

Gini after splitting on "Study Hours"

High: Yes=4, No=1  |  Low: Yes=2, No=3

Gini(High) = 1 − (4/5)² − (1/5)² = 1 − 0.64 − 0.04 = 0.320 Gini(Low) = 1 − (2/5)² − (3/5)² = 1 − 0.16 − 0.36 = 0.480 Weighted Gini = (5/10)×0.320 + (5/10)×0.480 = 0.160 + 0.240 = 0.400 Gini Reduction = 0.480 − 0.400 = 0.080
2

Gini after splitting on "Attendance"

Good: Yes=4, No=1  |  Poor: Yes=2, No=3

Gini(Good) = 1 − (4/5)² − (1/5)² = 0.320 Gini(Poor) = 1 − (2/5)² − (3/5)² = 0.480 Weighted Gini = (5/10)×0.320 + (5/10)×0.480 = 0.400 Gini Reduction = 0.480 − 0.400 = 0.080
3

Gini after splitting on "Sleep Hours"

Enough: Yes=4, No=2  |  Less: Yes=2, No=2

Gini(Enough) = 1 − (4/6)² − (2/6)² = 1 − 0.444 − 0.111 = 0.444 Gini(Less) = 1 − (2/4)² − (2/4)² = 1 − 0.250 − 0.250 = 0.500 ← worst possible Weighted Gini = (6/10)×0.444 + (4/10)×0.500 = 0.267 + 0.200 = 0.467 Gini Reduction = 0.480 − 0.467 = 0.013 ← barely any improvement

Key Intuition: Entropy vs Gini — What's Really Different?

Both measure the same underlying concept — impurity — using different math. Entropy borrows from information theory (Shannon, 1948) and uses logarithms. Gini borrows from probability and uses squaring.

Practically speaking: for most real datasets, they produce nearly identical trees. The choice almost never matters. Always use CART/Gini in production — it's faster. Use Entropy when learning theory or if you need to explain it academically.

Section 06
Building the Full Tree
recursive splitting · level by level

After we pick the best root feature, the algorithm repeats the exact same process on each branch — independently — until every leaf is pure or a stopping condition is met. This is called recursive splitting.

1

Level 1 — Root: Split on Study Hours

High branch rows 1,2,6,8,10 → Yes=4, No=1 — mostly pure (80% pass)

Low branch rows 3,4,5,7,9 → Yes=2, No=3 — still mixed, need to split further

2

Level 2 — Split each branch again using Attendance

High + Good (rows 1,2,8) → Yes=3, No=0 → ✔ Pure! Leaf = PASS

High + Poor (rows 6,10) → Yes=2, No=0 → ✔ Pure! Leaf = PASS

Low + Poor (rows 4,5,9) → Yes=0, No=3 → ✘ Pure! Leaf = FAIL

Low + Good (rows 3,7) → Yes=1, No=1 → Still mixed! Need one more split.

3

Level 3 — Split "Low + Good" using Sleep Hours

Low + Good + Enough (row 3) → Yes=1, No=0 → ✔ Leaf = PASS

Low + Good + Less (row 7) → Yes=0, No=1 → ✘ Leaf = FAIL

The Final Tree

All 10 students start here ↓ [Study Hours?] ┌────────────────────────────────────────┐ HIGH LOW [Attendance?] [Attendance?] ┌─────────────┐ ┌─────────────┐ Good Poor Good Poor ✔ PASS (3) ✔ PASS (2) [Sleep?] ✘ FAIL (3) ┌────────────┐ Enough Less ✔ PASS (1) ✘ FAIL (1)
📖 Reading the Tree

If you study High → you always pass, regardless of attendance or sleep (4 paths, all Yes).

If you study Low + Poor attendance → you always fail (3 students, 0 pass).

Sleep Hours only matters in 1 edge case: Low study + Good attendance.

⚠️ The Overfitting Problem

This tree memorizes our 10 training records perfectly. On new students, it might fail badly because it learned noise, not real patterns.

The Sleep Hours split (only 2 records!) is especially suspicious — this is exactly what Pruning fixes.

Section 07
Pre-Pruning
stop the tree from growing too deep · early stopping
✂️
Pre-Pruning = Set Limits BEFORE Training Starts

You define rules upfront. When the tree tries to split and violates a rule, it stops right there and makes that node a leaf. The tree never grows unnecessarily in the first place.

Why Pre-Prune?

Prevents overfitting before it happens. The tree never gets a chance to memorize noise or 1-record leaves.

How it works?

Set hyperparameters before fitting. During training, every time a split is considered, the conditions are checked first.

When to use?

Always as a starting point. Especially useful on large datasets where a full tree would be massive.

Pre-Pruning Techniques

1

max_depth — Limit how deep the tree can grow

What it does: Sets a hard ceiling on the number of question levels. A tree with max_depth=2 can only ask 2 questions on any path from root to leaf.

Effect on our tree: With max_depth=2, the Sleep Hours split at Level 3 would never happen. The "Low+Good" node (still mixed) would be forced into a leaf and predict the majority class (No).

Trade-off: Too small → tree misses real patterns (underfitting). Too large → tree memorizes noise (overfitting). Start with 3–5 and tune.

2

min_samples_split — Minimum records needed to allow a split

What it does: A node is only split if it has at least N records. If a node has fewer, it becomes a leaf (even if it's impure).

Effect on our tree: With min_samples_split=4, the "Low+Good" node (only 2 records) cannot be split — it becomes a leaf immediately.

Key insight: Splitting a 2-record node is usually memorizing noise, not learning a real pattern. This parameter stops that.

3

min_samples_leaf — Minimum records required in each resulting leaf

What it does: A split is only made if both resulting children would have at least N records.

Difference from min_samples_split: min_samples_split checks the parent. min_samples_leaf checks the children. If a proposed split would create a leaf with 1 record, the split is rejected.

Effect on our tree: With min_samples_leaf=2, the Sleep Hours split (which creates a 1-record leaf) is rejected.

4

min_impurity_decrease — Minimum gain required to justify a split

What it does: A split only happens if the impurity reduction is ≥ this threshold. If a feature doesn't reduce impurity enough, the split is skipped.

Effect on our tree: With min_impurity_decrease=0.05, Sleep Hours (IG = 0.020, Gini reduction = 0.013) would be rejected — both values are below 0.05. The tree would stop there.

This directly targets weak features like Sleep Hours in our dataset.

5

max_features — Limit how many features are considered at each split

What it does: At each split, only a random subset of features is evaluated — not all of them.

Why it helps: Reduces overfitting, adds diversity, and speeds up training. This randomness is what makes Random Forest work — each tree in the forest only sees a random subset of features.

Common values: 'sqrt' (√n features), 'log2' (log₂n features), or a fixed number.

Section 08
Post-Pruning
grow the full tree first · then cut back the weak parts
🌿
Post-Pruning = Grow Fully First, Then Trim Intelligently

Let the tree grow completely with no restrictions. Then go back and remove branches that don't actually improve performance on unseen data. It's smarter because you make pruning decisions with full information.

Why Post-Prune?

You see the full tree before deciding what to remove. Pruning decisions are based on actual validation accuracy — not guesses upfront.

How it works?

Grow full tree → evaluate subtrees on a held-out validation set → replace subtrees with leaves when it doesn't hurt accuracy.

When to use?

When you have enough data to reserve a validation set. When you want the most accurate pruning decision.

Post-Pruning Technique 1 — Reduced Error Pruning (REP)

The simplest and most intuitive post-pruning method. Works from the bottom of the tree upward.

1

Grow the full tree on training data

Allow the tree to grow completely — no depth limits, no sample restrictions. Our tree reaches depth 3 with the Sleep Hours split.

2

Pick the internal node closest to the leaves (bottom-up)

Start with the shallowest internal nodes — in our tree, that's the Sleep Hours node at the bottom. It's an internal node with 2 leaf children.

3

Replace that subtree with a single leaf (majority class)

The "Low + Good" group has Yes=1, No=1. Replacing it with a leaf → predict "No" (or "Yes", tie-broken arbitrarily). Evaluate accuracy on validation set.

4

Keep the pruning if validation accuracy doesn't decrease

If removing this branch gives same or better accuracy on the validation set → keep it pruned. A simpler tree with the same accuracy is always better (less overfitting).

5

Repeat bottom-up until no more improvements are possible

Move up the tree. Try pruning the Attendance nodes. Try pruning the Study Hours branches. Stop when no more beneficial pruning exists.

Post-Pruning Technique 2 — Cost-Complexity Pruning (CCP)

This is what scikit-learn implements via ccp_alpha. More principled and mathematically rigorous than REP.

🧮 The CCP Formula

CCP adds a complexity penalty to the tree's error. It penalizes every additional leaf node.

Cost(Tree, α) = Error(Tree) + α × NumberOfLeaves // Error(Tree) = misclassification rate (how wrong is the tree?) // NumberOfLeaves = total leaf nodes in the tree // α (alpha) = complexity penalty — you tune this // Higher α = bigger penalty for complexity = smaller tree // When α = 0 → no penalty → full tree is kept (no pruning) // When α = large → any extra leaf is heavily penalized → very small tree
1

Grow the full tree T₀ (α = 0, no penalty)

This is our full tree: depth 3, leaf nodes = 5 (3 pass leaves + 2 fail leaves + 2 bottom leaves).

2

Find the "weakest link" — the node where pruning is most justified

For each internal node, compute the α at which replacing it with a leaf would be equally good. The node with the smallest such α is the weakest link — prune it first.

In our tree: the Sleep Hours node (only 2 records, near-zero IG) will have the smallest α. It contributes almost nothing and costs 2 leaves.

3

Remove the weakest link → new tree T₁. Repeat.

T₀ (full, 5 leaves) → remove Sleep Hours split → T₁ (4 leaves) → remove next weakest → T₂ … → T_root (1 leaf, always predicts "Yes")

Each step produces a simpler tree. We now have a sequence: T₀, T₁, T₂, …

4

Use cross-validation to pick the best α

For each tree in the sequence, evaluate on a validation set. Pick the α (and its tree) that gives the best validation accuracy. That's your final pruned tree.

In sklearn: clf.cost_complexity_pruning_path(X_train, y_train) gives you all the α values. Cross-validate and choose the best.

Best Practice — Combine Both

Step 1: Start with pre-pruning. Set max_depth=4 and min_samples_leaf=3 to get a reasonable tree quickly.

Step 2: Then fine-tune using CCP post-pruning. Use cost_complexity_pruning_path() to find candidate alpha values, cross-validate, and pick the best one.

This two-step approach gives you both speed and accuracy.

Section 09
sklearn Code
everything in one place · copy-paste ready
⚠️ Important — What sklearn Uses Internally

sklearn's DecisionTreeClassifier always uses CART — binary splits, Gini by default. There is no ID3 in sklearn.

To get entropy (ID3-like metric), set criterion='entropy'. The splits will still be binary (CART-style), but using entropy to choose the best feature.

python 
# ─────────────────────────────────────────────────────────
# Decision Tree — Full Pipeline: Data → Train → Prune → Viz
# ─────────────────────────────────────────────────────────
from sklearn.tree import DecisionTreeClassifier, export_text, plot_tree
from sklearn.preprocessing import OrdinalEncoder
from sklearn.model_selection import cross_val_score
import numpy as np
import matplotlib.pyplot as plt

# ── STEP 1: Our dataset (strings need encoding)
X = [
    ['High', 'Good', 'Enough'],  # Row 1
    ['High', 'Good', 'Less'],    # Row 2
    ['Low',  'Good', 'Enough'],  # Row 3
    ['Low',  'Poor', 'Enough'],  # Row 4
    ['Low',  'Poor', 'Less'],    # Row 5
    ['High', 'Poor', 'Less'],    # Row 6
    ['Low',  'Good', 'Less'],    # Row 7
    ['High', 'Good', 'Enough'],  # Row 8
    ['Low',  'Poor', 'Enough'],  # Row 9
    ['High', 'Poor', 'Enough'],  # Row 10
]
y = ['Yes','Yes','Yes','No','No','Yes','No','Yes','No','Yes']

# sklearn needs numbers, not strings → encode
enc = OrdinalEncoder()
X_enc = enc.fit_transform(X)   # High→0, Low→1 etc. (auto)

# ── STEP 2: CART with Gini (sklearn default)
clf_gini = DecisionTreeClassifier(
    criterion='gini',          # CART — this is the default
    max_depth=3,               # PRE-PRUNE: max 3 levels deep
    min_samples_split=2,       # PRE-PRUNE: need at least 2 to split
    min_samples_leaf=1,        # PRE-PRUNE: each leaf needs ≥1 record
    random_state=42
)
clf_gini.fit(X_enc, y)

# ── STEP 3: ID3-like (entropy metric, binary splits)
clf_entropy = DecisionTreeClassifier(
    criterion='entropy',       # Entropy (ID3-style metric)
    max_depth=3,
    random_state=42
)
clf_entropy.fit(X_enc, y)

# ── STEP 4: POST-PRUNING with Cost-Complexity Pruning (CCP)

# 4a. Get all candidate alpha values (the pruning path)
path = clf_gini.cost_complexity_pruning_path(X_enc, y)
ccp_alphas = path.ccp_alphas[:-1]  # remove last (trivial)

# 4b. Train one tree per alpha value
pruned_trees = []
for alpha in ccp_alphas:
    clf = DecisionTreeClassifier(ccp_alpha=alpha, random_state=42)
    clf.fit(X_enc, y)
    pruned_trees.append(clf)

# 4c. Cross-validate to find the best alpha (on larger datasets)
cv_scores = [
    cross_val_score(clf, X_enc, y, cv=5).mean()
    for clf in pruned_trees
]
best_alpha = ccp_alphas[np.argmax(cv_scores)]
best_clf = pruned_trees[np.argmax(cv_scores)]

print(f"Best alpha: {best_alpha:.4f}")
print(f"Best CV score: {max(cv_scores):.3f}")

# ── STEP 5: Visualize the tree as text
feature_names = ['StudyHrs', 'Attendance', 'SleepHrs']
print(export_text(clf_gini, feature_names=feature_names))

# ── STEP 6: Predict a new student
new_student = enc.transform([['High', 'Poor', 'Less']])
prediction = clf_gini.predict(new_student)
print(f"Prediction: {prediction[0]}")  # Output: Yes
✅ Quick Debugging Guide

Tree memorizing training data (100% train acc, low test acc)? → Reduce max_depth or increase min_samples_leaf

Tree performing poorly on both train and test (underfitting)? → Increase max_depth, reduce min_samples_split

Want optimal automated pruning? → Tune ccp_alpha with cross_val_score

Gini vs Entropy — which to choose? → Both give nearly identical results. Gini is faster.

Everything You Learned

The complete mental model — all 9 sections in 8 cards

Core Idea

Recursively split data by the feature that creates the purest (least mixed) child groups. Repeat until leaves are pure or stopping rules trigger.

ID3 — Entropy

Measures disorder using logarithms. Picks feature with highest Information Gain. Multi-way splits. Academic use only — not in sklearn.

CART — Gini

Measures disorder using squaring. Always binary splits. Supports both classification and regression. Used inside sklearn — this is your go-to.

What We Found

Both algorithms agree: Study Hours and Attendance are the best root features (IG=0.124, Gini↓=0.080). Sleep Hours is nearly useless (IG=0.020).

Pre-Pruning

Stop growth during training. Use: max_depth, min_samples_leaf, min_samples_split, min_impurity_decrease. Fast. Always try first.

Post-Pruning

Grow fully then trim. REP: bottom-up leaf replacement tested on validation set. CCP: use ccp_alpha in sklearn, tune with cross-validation.

When to Choose

Pre-pruning → first pass, fast, large data. Post-pruning (CCP) → when you have a validation set and want the most accurate simplification.

sklearn Defaults

criterion='gini' (CART). Set criterion='entropy' for entropy. Use ccp_alpha for post-pruning. DecisionTreeClassifier is always CART internally.
